# 周报总结技能

## 技能说明

此技能用于快速总结周报的核心内容，输出简洁的结构化摘要，并原封不动更新日报模板的待办任务。完成后会询问用户是否要上传到 GitHub。

## 使用方法

用户提供周报文件的完整路径，技能将自动读取并总结，然后将下周计划原封不动覆盖到日报模板，最后询问是否上传到 GitHub。

## 执行步骤

### 1. 读取周报文件
使用 Read 工具读取用户指定的文件路径。

### 2. 提取关键信息

从文件中提取以下三类信息：

#### 完成的任务
- 从「待办任务」中找出标记为 `[x]` 已完成的任务
- 从「开发任务」中找出带有日期标记为"今天"或"已完成"的任务
- 从「今日收获」中提取技术收获

#### Bug 解决情况
- 从「bug解决」中找出有「解决方案」或「验证结果」的 bug
- 记录 bug 编号和简要解决方案

#### 下周/明日计划
- 从「待办任务」中找出未完成的任务
- 从「明日安排」中提取具体计划
- 从「下周计划」中提取具体计划

### 3. 输出总结

使用以下 Markdown 格式输出：

```markdown
## 周报总结

### 完成的任务
- [任务1] ✅
- [任务2] ✅

### Bug 解决情况
- **[BUG编号]** [简要描述]: [解决方案]
- ...

### 下周/明日计划
- [待办任务1]
- [待办任务2]
```

### 4. 更新日报模板（重要）

**必须执行**：将提取的下周计划**原封不动**覆盖到日报模板的「待办任务」部分。

**模板路径**: `D:\code\文档\模板\日报模板-简洁版.md`

**操作步骤**:
1. 读取日报模板文件
2. 找到 `## 待办任务` 部分（从 "## 待办任务" 到下一个 "##" 标题之前）
3. **原封不动**地将从周报提取的下周计划内容替换到该区域
4. **不要做任何修改**：
   - 不要删除已完成的任务
   - 不要合并或拆分任务
   - 不要更改任务描述
   - 保持原有的格式和缩进

**示例**:
```markdown
# 周报中的下周计划（原始内容）
- [ ] 任务A
- [ ] 任务B
  - [ ] 子任务B1
- [ ] 任务C

# 更新到日报模板后（完全一致，不做任何改动）
- [ ] 任务A
- [ ] 任务B
  - [ ] 子任务B1
- [ ] 任务C
```

### 5. 询问是否上传到 GitHub

**必须执行**：在完成所有步骤后，询问用户是否需要上传到 GitHub。

**询问方式**：
```
---

📦 是否需要上传到 GitHub？

请回复：
- "是" 或 "上传" → 执行上传
- "否" 或 "不上传" → 跳过上传
```

**等待用户回复后**：
- 如果用户确认「是」→ 调用 git-upload skill 执行上传
- 如果用户选择「否」→ 结束技能，输出完成信息

## 输出要求

- 每项内容保持简短，一句话概括
- 使用项目符号列表
- Bug 编号使用加粗格式
- 保持格式统一规范
- **必须原封不动覆盖日报模板，不要做任何改动**
- **必须询问用户是否上传 GitHub，得到确认后才能执行上传**

## 注意事项

- 如果某个部分没有内容，该部分可以省略
- 如果文件不存在或无法读取，提示用户检查路径
- 优先提取明确标记为"已完成"的任务，避免误判
- **日报模板更新失败时，必须向用户报告错误**
- **重要：只需要覆盖内容，不要修改、删除或添加任何任务**
- **必须等待用户明确确认后才能执行 GitHub 上传**
