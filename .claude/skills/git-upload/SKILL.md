# Git 文档上传技能

## 技能说明

此技能用于将本地改动的文件上传到 GitHub 仓库。支持选择性地上传文件，并自动生成规范的 commit 信息。

## 使用方法

用户触发 skill 后，会自动列出所有改动的文件，用户选择要上传的文件后执行上传。

## 执行步骤

### 1. 检测改动文件

使用 Bash 工具执行 git status 命令，列出所有有改动的文件：

```bash
git status
```

**输出包括**：
- `Changes not staged for commit` - 已修改但未暂存的文件
- `Untracked files` - 未跟踪的新文件
- `Changes to be committed` - 已暂存的文件

### 2. 展示改动文件列表

将所有改动文件整理成清晰的列表，使用编号展示：

```
📁 待上传文件列表：

已修改文件：
1. 文件名1.md
2. 文件名2.md

新增文件：
3. 新文件1.md
4. 新文件2.md
```

### 3. 用户选择文件

**询问用户**：请告诉我需要上传的文件编号（如：1,3,4 或 all 上传全部）

### 4. 处理用户选择

- 如果用户输入编号 → 只 add 用户选择的文件
- 如果用户输入 `all` → add 所有改动文件
- 如果用户输入文件名 → 从列表中移除该文件后，重新询问

### 5. 为每个文件生成 commit 信息

对于每个要上传的文件，读取文件内容并生成 commit 信息：

**生成规则**：
1. 如果文件是 Markdown（.md）：
   - 读取文件第一行作为标题（# 标题 或 --- 之间的内容）
   - 如果有 YAML frontmatter，尝试从 title 字段获取
2. 如果是其他类型文件：
   - 使用文件名作为 commit 信息基础

**commit 信息格式**：
```
[文件类型]: [文件名/标题]

- 更新日期: YYYY-MM-DD
- 自动提交

Co-Authored-By: Claude Haiku 4.5 <noreply@anthropic.com>
```

### 6. 执行提交和推送

**必须执行的操作**：
```bash
# 添加用户选择的文件
git add "文件路径1" "文件路径2" ...

# 提交
git commit -m "commit信息"

# 推送
git push origin main
```

**处理推送失败**：
如果遇到远程有更新导致推送被拒绝：
1. 先 stash 本地更改
2. 执行 `git pull --rebase origin main`
3. 执行 `git push origin main`
4. 恢复本地更改 `git stash pop`

### 7. 显示上传结果

展示最终结果：
```
✅ 文件上传成功！

已上传文件：
- 文件1.md
- 文件2.md

提交信息：xxx
提交哈希：xxx
仓库地址：github.com:xxx
```

## 输出要求

- 改动文件列表必须清晰分类（已修改/新增）
- 必须等待用户确认后再执行上传
- commit 信息必须包含日期和 Co-Authored-By
- 必须处理推送失败的情况

## 注意事项

- 如果没有改动文件，提示用户"没有需要上传的文件"
- 如果仓库没有初始化，提示用户先初始化 Git
- 如果用户选择文件后列表为空，提示用户"没有选择任何文件"
- 必须等待用户确认后再执行 add 操作
