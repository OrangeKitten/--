## 联调
dsp 联调 IAudioControl功能都要写在一个aidl文件中，否则德文那边不能初始化。并且AOSP提供的也是这样写的2026-03-30

**结论：**
使用AIDL的callback 要继承bnXXX如下
```
/**

 * AIDL callback adapter

 * Converts AIDL callbacks to internal ICarAudioHalCallback

 */

class GxaAidlCallback : public ::aidl::vendor::idc::hardware::dsp::BnAudioControlCallback

{

public:

    GxaAidlCallback(sp<ICarAudioHalCallback> callback)

        : pCallback(callback) {}
  
    virtual ~GxaAidlCallback() {}

    ::ndk::ScopedAStatus onReceiveSourceRequester(

        const ::aidl::vendor::idc::hardware::dsp::types::source_requester& in_requester) override;

    ::ndk::ScopedAStatus onReceiveEvent(

        const ::aidl::vendor::idc::hardware::dsp::types::audio_event& in_event) override;

    ::ndk::ScopedAStatus onSetParameters(const std::string& in_value) override;

private:

    sp<ICarAudioHalCallback> pCallback;

};
```
不能继承ICarAudioHalCallback，否则传给server的callback对象会是空指针。

## 今日收获
![[img_v3_02109_a63c3161-5f9d-4e12-a201-da8ee8274fag.jpg]]



蓝牙音乐跟mute焦点的仲裁是C，原因如下：
车机跟手机都是可以调节音量的前提是蓝牙音乐持有焦点，如果跟MUTE仲裁是R那么蓝牙音乐就会失去焦点不能调节音量了。
蓝牙音乐是主动监听焦点mute状态的

## 三、关键发现

1. 所有OMX软件编解码器均已迁移到Codec 2.0
2. IAMF是存根实现，实际不可用（返回C2_BAD_STATE）
3. APV默认禁用，需minsdk≥36
4. AV1有3套解码实现：libgav1、dav1d、aom(已弃用)
5. 组件注册入口：C2Store.cpp 硬编码所有.so库列表
